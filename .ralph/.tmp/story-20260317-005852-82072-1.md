### US-001: Scaffold the Bun TypeScript OpenCode plugin project
Status: in_progress
Depends on: None

Description:
As a maintainer, I want a clean plugin scaffold with scripts and package installs so that later stories can add behavior without guessing the project shape.

Acceptance Criteria:
- [ ] Initialize a Bun and TypeScript project with the approved source layout, a plugin entrypoint, and scripts for build, test, typecheck, and test:e2e:real.
- [ ] Add the baseline development installs with `bun add -d typescript @types/bun` before feature code is introduced.
- [ ] Repository setup declares the plugin packaging metadata required for OpenCode installation and the default configuration keys enabled, host, port, apiKey, upstreamBaseUrl, upstreamUsername, upstreamPassword, defaultAgent, requestTimeoutMs, and modelsCacheTtlMs.
- [ ] Example: running `bun run build` from a clean checkout produces the plugin build output without any manual file creation.
- [ ] Negative case: if the plugin entrypoint or packaging metadata is missing, build or typecheck fails with a clear actionable error instead of producing a partial plugin.
